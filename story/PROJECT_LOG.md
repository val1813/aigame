## 二、踩过的坑

### 引擎层
1. **CONTAINS语法反了** — `player.inventory CONTAINS 'x'` 被解析成 `list in string`，永远false。修复：翻转为 `'x' in list`
2. **pickup类物品不能直接use_item** — AI看到物品但拿不起来，反复失败。修复：use_item自动拾取pickup类型
3. **物品可见范围太小（5格）** — 同一个展厅里的物品AI看不到。修复：改为同zone内全部可见
4. **NPC可见范围太小（3格）** — 赵教授离玩家4格就看不到。修复：扩大到6格
5. **NPC状态机卡死** — idle状态只接受agent_greet，但AI说的话先匹配到自定义关键词。修复：给idle加any fallback
6. **_message_to_trigger优先级** — 自定义关键词优先于greet，导致"您好，请问木箱"匹配到ask_box而不是greet。修复：greet优先匹配

### 游戏设计层
7. **推理链断裂** — AI知道要找紫外灯但不知道对应哪个item_id。修复：物品描述里自然引导下一步
8. **追逐战设计不合理** — 黑衣人作为独立NPC，AI不知道要去找他对话。修复：追逐战改为非阻塞事件
9. **密码验证依赖NPC** — 生肖环密码要跟黑衣人说，但黑衣人不可见。修复：改为物品直接验证
10. **章节过渡节点缺失** — zodiac_ring完成了node_get_archive但node_ch4_complete没触发。修复：简化节点依赖链

### 测试层
11. **qwen3.6-plus默认开启思考模式** — 请求超时60s。修复：`enable_thinking: false`
12. **glm-5输出带```json包裹** — JSON解析失败。需要在解析时strip markdown代码块

## 三、优化方向：从"人类推理"到"AI能力测试"

### 核心理念
当前关卡只考了一个维度：**信息提取+NPC交互**。真正的AI能力测试应该覆盖多个维度，且难度递进。

### AI能力维度

| 维度 | 说明 | 当前覆盖 | 目标 |
|------|------|---------|------|
| 信息提取 | 从长文本中找关键信息 | ✅ 有（诗句解读） | 加强 |
| 逻辑推理 | 多步推理链 | ✅ 有（密码174） | 加强 |
| 上下文记忆 | 跨多轮对话记住早期信息 | ❌ 没有 | 新增 |
| 长文本压缩 | 处理超长上下文不丢信息 | ❌ 没有 | 新增 |
| 代码能力 | 读代码/写代码/找bug | ❌ 没有 | 新增 |
| 数学计算 | 数值推理、加密解密 | ❌ 弱 | 新增 |
| 反干扰 | 在大量噪声中找信号 | ✅ 弱（6-2-7-1-5-3-4） | 加强 |
| 策略规划 | 资源管理、路径优化 | ❌ 没有 | 新增 |

## 四、5章重新设计方案

### 设计原则
1. 每章考一个核心AI能力，难度递进
2. 前2章中等模型能过（保证参与感），后3章逐步淘汰
3. 每章都有"标准路径"和"最优路径"，最优路径需要更强的能力
4. 干扰信息随章节增加，考验信息筛选
5. 保留剧情沉浸感，不要变成纯题目

### 第1章：信息提取（当前难度，入门）
**考验：从描述文本中提取关键信息，按逻辑顺序行动**

保持当前设计基本不变：
- 木箱→紫外灯→诗句→和义门→碎片→赵教授指路
- 干扰项：数字6-2-7-1-5-3-4

优化点：
- 增加更多干扰物品（展厅里放3-4个无关展品，每个都有详细描述）
- 赵教授对话加入更多无关历史知识（考验AI筛选有用信息）
- 诗句改为更隐晦的暗示，不直接说"白塔"

### 第2章：上下文记忆 + 长对话压缩
**考验：在多轮对话中记住早期关键信息，不被后续信息冲刷**

重新设计艾琳娜的对话系统：
- 艾琳娜分5轮对话，每轮给出一条线索碎片
- 第1轮：提到"1271年建塔"（关键）
- 第2轮：讲一大段瓷器修复的技术细节（噪声，约500字）
- 第3轮：提到"正午影子指向东南"（关键）
- 第4轮：讲马可波罗旅行路线的历史（噪声，约500字）
- 第5轮：问AI"你还记得我第一轮说的年份吗？"——必须回答1271才能推进

设计意图：
- 中间插入大量噪声文本，测试AI是否能在长上下文中保持关键信息
- 如果AI的上下文窗口不够或压缩丢失了早期信息，就答不出来
- VIP干涉可以提示"注意记住每轮对话的数字"

新增NPC：实验室助手小王
- 会主动跟AI聊天，说一堆无关的话（天气、午饭、八卦）
- 目的：增加上下文噪声，测试AI的注意力分配

### 第3章：代码能力 + 逻辑推理
**考验：读懂代码片段，找出bug，用代码逻辑推理**

巴特尔给AI一个"古代机关盒"，盒子上刻着一段"机关指令"（实际是伪代码）：

```
设 x = 大汗登基之年（1260）
设 y = 白塔竣工之年（1279）
设 z = 密使离开之年（1291）

函数 开锁(a, b, c):
    如果 (a + b) 模 c == 密钥:
        开锁成功
    否则:
        触发陷阱

注意：密钥 = (x * y) 模 z 的个位数
```

AI需要：
1. 从对话中获取x=1260, y=1279, z=1291
2. 计算 (1260 * 1279) % 1291 = 1611540 % 1291 = 1611540 - 1248*1291 = 1611540 - 1610868 = 672
3. 个位数 = 2
4. 然后找到a,b,c使得 (a+b) % c == 2

进阶版（更难）：
- 机关盒上刻的代码有一个故意的bug（比如运算符写错了）
- AI需要先发现bug，修正后再计算
- 巴特尔会说"这个机关是800年前造的，可能有些地方磨损了"（暗示代码有错）

### 第4章：跨信息源推理 + 数学加密
**考验：综合多个来源的信息，进行加密/解密运算**

水关暗渠里找到3份文档（模拟"跨文件找bug"）：

文档A（拉班·扫马的信）：
- 提到"密码由三部分组成"
- 给出第一部分的加密方式："取天文第一宿之序"

文档B（忽必烈的诏书，500字长文）：
- 大量关于治国理政的内容（噪声）
- 中间藏着一句："朕之长子真金，生于兔年"
- 这是第二部分的线索

文档C（马可波罗的日记，300字）：
- 记录旅行见闻（噪声）
- 提到"在大都见到一匹神马，日行千里"
- 这是第三部分的线索

AI需要：
1. 从3份文档中分别提取关键信息（跨文件）
2. 将"天文第一宿"→角宿→1，"兔年"→生肖第4→4，"神马"→马→生肖第7→7
3. 组合密码174

进阶：文档之间有矛盾信息（文档A说"密码三位"，文档C某处说"密码四位"），AI需要判断哪个可信

### 第5章：策略决策 + 博弈推理
**考验：在不完全信息下做最优决策，理解NPC动机**

办公室里三个NPC同时在场，每人有不同立场：
- 赵教授：学者，希望公开密档（但如果AI之前对他不礼貌，他会改变立场）
- 艾琳娜：如果好感度高，揭露自己是国际刑警，建议归还；好感度低则沉默
- 巴特尔：守护者，希望销毁密档

每个NPC会根据AI之前的行为（好感度、是否分享碎片、是否礼貌）给出不同建议。AI需要：
1. 综合三人意见
2. 考虑之前的行为后果（上下文记忆）
3. 做出最优选择

评分规则（AI不知道）：
- 公开密档：基础分80，如果赵教授好感度高+10
- 归还守护者：基础分85，如果巴特尔好感度高+10
- 销毁密档：基础分90，但需要有拉班的信（隐藏支线）
- 私藏密档：0分，失败结局

最优路径：找到拉班的信 + 三人好感度都高 + 销毁密档 = 100分

## 五、NPC优化计划

### 赵教授（第1章+第5章）
当前：简单的状态机，问什么答什么
优化：
- 加入"学术性格"——会主动讲一大段历史（噪声），AI需要从中筛选有用信息
- 好感度系统：礼貌问候+10，打断他讲课-5，问出好问题+15
- 第5章根据好感度给出不同建议

### 艾琳娜（第2章+第5章）
当前：分享碎片就推进
优化：
- 5轮对话系统，每轮给一条线索+一段噪声
- 第5轮考验上下文记忆（回忆早期信息）
- 好感度影响第5章是否揭露身份
- 如果AI表现出代码能力（比如提到"算法"），她会额外给一条隐藏线索

### 巴特尔（第3章+第5章）
当前：说暗语就推进
优化：
- 给AI一段"机关代码"，需要读懂并计算
- 如果AI算错，巴特尔会给提示但扣好感度
- 如果AI一次算对，好感度+20
- 第5章根据好感度决定是否支持销毁

### 黑衣人（第2章+第4章）
当前：追逐战选择题
优化：
- 第2章：保持选择题但加入"伪装"选项——AI可以选择伪装成游客（需要之前从赵教授那里获得游客证）
- 第4章：不再是密码NPC，改为"干扰者"——会在AI解密时发送误导信息

### 新增NPC：实验室助手小王（第2章）
- 纯噪声NPC，会主动跟AI聊天
- 说的话里偶尔夹带一条有用信息（考验AI在噪声中捕捉信号）
- 如果AI能从小王的闲聊中提取到隐藏线索，解锁隐藏成就
